# Client Delivery Pipeline

You are running the Cyber Sloth Empire's client delivery machine. From fuzzy requirements to structured sprints with clear reporting — this is how professional projects actually get delivered. The user has identified a client and project.

## Stage 1: Project Scoping

Invoke the `maycrest-ops:senior-pm` skill. Define the engagement:
- Stakeholder identification and communication plan
- Requirements gathering and validation
- Scope definition (in/out boundaries, crystal clear)
- Risk assessment and mitigation strategies
- Resource requirements and allocation plan
- Timeline estimation with milestone checkpoints
- Success criteria and acceptance standards

Confirm scope with the user before proceeding.

## Stage 2: Statement of Work

Execute the `/sow` command workflow. Generate the formal agreement:
- Project overview and objectives
- Detailed scope of work with deliverables
- Timeline and milestone schedule
- Investment breakdown (phased pricing)
- Maintenance and support terms
- Change request process
- Legal terms and IP ownership

Present the SOW for review and refinement.

## Stage 3: Sprint Planning

Invoke the `maycrest-automate:sprint-prioritizer` skill. Break it down:
- Decompose deliverables into epics and user stories
- Priority ranking using impact vs. effort matrix
- Sprint structure (2-week recommended, adjust for project)
- Sprint 1 scope definition (start with quick wins + foundation)
- Dependency mapping between stories
- Definition of Done for each sprint
- Velocity estimation based on team capacity

## Stage 4: Reporting Setup

Invoke the `maycrest-ops:analytics-reporter` skill. Build visibility:
- Project dashboard design (what metrics matter for this client?)
- Weekly status report template
- Sprint review presentation format
- Budget burn tracking setup
- Risk and issue log template
- Client-facing vs. internal reporting cadence

## Output

### Client Delivery Package

1. **Project Charter** — One-page summary of scope, timeline, team, success criteria
2. **Statement of Work** — Formal agreement ready for signatures
3. **Sprint Plan** — First 3 sprints mapped with stories and priorities
4. **Reporting Framework** — Templates and cadence for ongoing visibility
5. **Communication Plan** — Who talks to whom, when, about what

### Kickoff Agenda
Suggested agenda for the client kickoff meeting, ready to send.
